A website counting travel carbon emissons

This is a team work done at University of Bristol. Our team used agile development to create a dashboard that helps companies analyze their travel carbon emissions and track progress towards carbon neutrality. We developed the back-end using Spring Boot and hosted it on Google Cloud, with Firebase for both our database and front-end hosting. This efficient setup allowed us to build a user-friendly platform that contributes positively to environmental sustainability. Responding to our client RouteZero's specific requirements, we integrated a line chart feature within the dashboard. This tool enables users to view and compare their carbon emissions on a weekly, monthly, and annual basis. Additionally, we developed a detailed page that breaks down emissions by type of transportation, helping users identify the most significant sources of carbon output. This tailored functionality enhances the dashboard's utility in strategic planning and tracking of environmental impact goals.

A Muti-agent maze solver in Prolog

This is individual work. I developed an algorithm that uses multiple agents to explore a initially hidden maze concurrently. These agents traverse the maze greedily, uncovering its layout until one of them discovers the exit. The maze is structured with black tiles representing impassable walls, green tiles indicating navigable paths and grey tiles are unexplored. Any other colored tiles within the maze represents the agent that has previously explored that area. This visual setup helps in tracking the agents’ paths and their progress towards solving the maze. The project not only demonstrates my proficiency in Prolog but also showcases my ability to implement complex algorithms for dynamic problem-solving. Press the button for a demo.

Game of Life
 golang

This is a pair programming unit. The British mathematician John Horton Conway devised a cellular automaton named ‘The Game of Life’. The game resides on a 2-valued 2D matrix, i.e. a binary image, where the cells can either be ‘alive’ (white) or ‘dead’ (black). We first wrote code to evolve Game of Life using multiple worker goroutines on a single machine. Secondly, we created an implementation that uses a number of AWS nodes to cooperatively calculate the new state of the Game of Life board, and communicate state between machines over a network.
